# SPEC-002: 出退勤打刻

## 概要
作業員がW-01（打刻画面）で出勤・退勤ボタンを押し、勤務ステータスを変更する。
内部的には時刻を記録するが、作業員には時刻・勤務時間を表示しない。
給与に関わる時間管理は管理者がA-01で行う。
管理者も「自分の打刻」タブ（A-01内）から同様の操作が可能。

## 対応画面
- W-01: 打刻画面（ホーム）
- A-01: 管理ダッシュボード（「自分の打刻」タブ）

## 対応ユースケース・ルール
- UC-01: 出勤を打刻する
- UC-02: 退勤を打刻する
- 境界コンテキスト「勤怠管理」ルール#1〜#15

---

## 受入基準

### 出勤
- [ ] [出勤] ボタンを押すと、現在時刻がTimeStampとして内部的に保存される（作業員には非表示）
- [ ] 保存後、ステータスが「未出勤」→「勤務中」に変わる
- [ ] 画面上のステータス表示が更新される
- [ ] 出勤ボタンが非活性化され、退勤・現場開始ボタンが活性化される
- [ ] 既に出勤済みの場合、出勤ボタンは非活性（押せない）
- [ ] GPS位置情報の取得を試みる。取得できなくても打刻は成功する

### 退勤
- [ ] [退勤] ボタンを押すと、現在時刻がTimeStampとして内部的に保存される（作業員には非表示）
- [ ] 保存後、ステータスが「勤務中」→「退勤済み」に変わる
- [ ] 勤務時間は内部的に計算されるが、作業員には表示しない（管理者がA-01で確認）
- [ ] 現場作業中に退勤しようとした場合、先に現場終了を促すメッセージが表示される
- [ ] 未提出の作業報告がある場合、確認ダイアログを表示する
  - 「報告してから退勤」→ W-03（作業報告画面）へ遷移
  - 「このまま退勤」→ 退勤処理を実行
- [ ] 未出勤の場合、退勤ボタンは非活性（押せない）

### 共通
- [ ] 管理者もworkerと同様に自分の出退勤を打刻できる（A-01の打刻タブ）
- [ ] 共有端末でも、ログインユーザーとして打刻される
- [ ] オフラインでも打刻でき、接続回復時にデータが保持される

## 状態遷移

```
  [未出勤]
      │
      │ 出勤ボタン（ClockIn）
      ↓
  [勤務中] ←───── 現場終了（SiteEnd） ←── [現場作業中]
      │                                      ↑
      │ 退勤ボタン（ClockOut）               │
      │                                      │ 現場開始（SiteStart）
      ↓                                      │
  [退勤済み]                             ※ SPEC-003で定義
```

### 各状態で有効なボタン

| 状態 | 出勤 | 退勤 | 現場開始 | 現場終了 |
|------|:---:|:---:|:------:|:------:|
| 未出勤 | ✅ | ❌ | ❌ | ❌ |
| 勤務中 | ❌ | ✅ | ✅ | ❌ |
| 現場作業中 | ❌ | ⚠️※ | ❌ | ✅ |
| 退勤済み | ❌ | ❌ | ❌ | ❌ |

> ※ 現場作業中に退勤しようとした場合は「先に現場終了してください」と表示

## 操作フロー

### 出勤（正常系）

| # | ユーザー操作 | システム動作 |
|---|------------|------------|
| 1 | [出勤] ボタンをタップ | — |
| 2 | — | GPS位置情報の取得を試みる |
| 3 | — | TimeStamp { type: 'ClockIn', time: now(), location } を保存 |
| 4 | — | DailyRecord.status を 'working' に更新 |
| 5 | — | 「出勤しました」トースト表示（時刻は含めない） |
| 6 | — | ボタン状態を更新（出勤→非活性、退勤/現場開始→活性） |

### 退勤（正常系）

| # | ユーザー操作 | システム動作 |
|---|------------|------------|
| 1 | [退勤] ボタンをタップ | — |
| 2 | — | 未提出の作業報告をチェック |
| 3 | （未提出なし） | TimeStamp { type: 'ClockOut', time: now(), location } を保存 |
| 4 | — | DailyRecord.status を 'finished' に更新 |
| 5 | — | 勤務時間を内部的に計算（作業員には非表示） |
| 6 | — | 「退勤しました。お疲れ様でした」トースト表示 |

---

## エラーハンドリング

| 条件 | 動作 | メッセージ |
|------|------|-----------|
| 既に出勤済み | 出勤ボタン非活性 | — |
| 未出勤で退勤 | 退勤ボタン非活性 | — |
| 現場作業中に退勤 | 警告ダイアログ | 「現場作業中です。先に現場終了してください」 |
| 未提出報告ありで退勤 | 確認ダイアログ | 「未提出の作業報告があります。退勤しますか？」 |
| GPS取得失敗 | location = null で打刻続行 | — |

## データモデル

```javascript
// 打刻タイプ
TimeStampType = 'ClockIn' | 'ClockOut' | 'SiteStart' | 'SiteEnd'

// 勤務ステータス
WorkStatus = 'notStarted' | 'working' | 'onSite' | 'finished'

// 打刻レコード
TimeStamp {
  type: TimeStampType,
  time: string,              // ISO8601（例: "2026-07-21T09:00:15+09:00"）
  location: Location | null,
  jobId: string | null       // SiteStart/SiteEnd時のみ使用（SPEC-003）
}

// 位置情報
Location {
  lat: number,
  lng: number
}

// 日次記録
DailyRecord {
  workerId: string,
  date: string,              // "2026-07-21"
  status: WorkStatus,
  timeStamps: TimeStamp[],
  confirmedHours: number | null  // 管理者が確定した勤務時間（SPEC-004）
}
```

## UI仕様（W-01 打刻画面）

### レイアウト構成
- **ヘッダー**: アプリ名、[履歴] / [設定] アイコン
- **ステータス表示**: 「未出勤」「勤務中」「現場作業中（○○マンション）」「退勤済み」（大きく表示）
- **打刻ボタン群**: 出勤 / 退勤 / 現場開始 / 現場終了（状態に応じて活性/非活性）
- **未提出バッジ**: 未提出の作業報告がある場合に表示

> ⚠️ **作業員向け画面には以下を表示しない**（勤怠管理ルール#15）:
> - 現在時刻（リアルタイム）
> - 出退勤時刻
> - 勤務時間
> - 本日の打刻履歴（時刻付き）
>
> 理由: 細かい時刻を見せると給与計算とのズレで不信を招くため。
> 時間に関する管理は管理者がA-01ダッシュボードで行う。

## スコープ外
- 現場開始/終了の詳細（→ SPEC-003 で定義）
- 作業報告の詳細（→ Phase 2 SPEC-006 で定義）
- 打刻修正機能（管理者がA-01で確定勤務時間を入力する運用で代替）
