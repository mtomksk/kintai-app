# SPEC-001: ログイン認証

## 概要
ユーザーがID・パスワードでログインし、ロール（worker / admin）に応じた画面に遷移する。

## 対応画面
- S-01: ログイン画面

## 対応ユースケース・ルール
- 境界コンテキスト「認証・アカウント」ルール#1〜#9

---

## 受入基準

- [ ] ユーザーIDとパスワードを入力してログインできる
- [ ] worker ロールでログイン → W-01（打刻画面）に遷移する
- [ ] admin ロールでログイン → A-01（管理ダッシュボード）に遷移する
- [ ] パスワードが間違っている場合、エラーメッセージが表示される
- [ ] ユーザーIDが存在しない場合、エラーメッセージが表示される
- [ ] ログイン成功後、セッション（またはlocalStorage）にユーザー情報が保存される
- [ ] 共有端末でログアウト→別アカウントでログインが可能
- [ ] ログイン画面にパスワード変更機能は表示しない（管理者がA-06で管理）
- [ ] 表示言語はアカウントの `language` 設定に従う

---

## 操作フロー（正常系）

| # | ユーザー操作 | システム動作 |
|---|------------|------------|
| 1 | ユーザーIDを入力 | — |
| 2 | パスワードを入力 | — |
| 3 | [ログイン] ボタンをタップ | — |
| 4 | — | Account データから userId を検索 |
| 5 | — | パスワードを照合 |
| 6 | — | セッションにユーザー情報を保存（userId, role, language, name） |
| 7 | — | role に応じた画面に遷移 |

## エラーハンドリング

| 条件 | 動作 | メッセージ |
|------|------|-----------|
| ユーザーIDが空 | ログインボタン非活性 or エラー表示 | 「ユーザーIDを入力してください」 |
| パスワードが空 | ログインボタン非活性 or エラー表示 | 「パスワードを入力してください」 |
| ユーザーID不一致 | エラーメッセージ表示 | 「ユーザーIDまたはパスワードが正しくありません」 |
| パスワード不一致 | エラーメッセージ表示 | 「ユーザーIDまたはパスワードが正しくありません」 |

> セキュリティのため、IDとパスワードどちらが間違っているかは区別しない。

---

## データモデル

```javascript
Account {
  userId: string,       // ログインID（例: "worker01"）
  password: string,     // パスワード（平文 → 将来ハッシュ化）
  role: 'worker' | 'admin',
  name: string,         // 表示名（例: "田中太郎"）
  language: 'ja' | 'si' | 'en',
  teamId: string | null // 所属チーム（任意）
}
```

```javascript
// ログイン後にセッション/localStorageに保存する情報
Session {
  userId: string,
  role: 'worker' | 'admin',
  name: string,
  language: 'ja' | 'si' | 'en'
}
```

---

## バリデーション

| ルール | 条件 | 元ネタ |
|--------|------|--------|
| 英数字パスワード | パスワードは英数字のみ | 認証 ルール#1 |
| 1:1紐づけ | 1ユーザー = 1アカウント | 認証 ルール#2 |

---

## UI仕様（S-01）

- モバイルファーストのシンプルなフォーム
- ユーザーID入力フィールド
- パスワード入力フィールド（マスク表示、トグルで表示/非表示切替）
- [ログイン] ボタン
- エラーメッセージ表示エリア
- 「パスワードを忘れた場合」のリンクは **表示しない**

---

## スコープ外
- パスワードのハッシュ化（Phase 3以降のバックエンド導入時に対応）
- パスワードリセット機能（管理者がA-06で直接変更する運用）
- 二要素認証
- セッションの有効期限管理
