# SPEC-003: 現場作業管理

## 概要
出勤後の作業員が、案件を選択して現場作業を開始・終了する。
案件選択画面（W-02）で当日の担当案件を確認し、鍵情報やGoogleマップで現場に向かい、
打刻画面（W-01）で現場開始/終了を記録する。
1日に最大5件の現場を担当する可能性がある。

## 対応画面
- W-01: 打刻画面（現場開始/終了ボタン）
- W-02: 案件選択画面

## 対応ユースケース・ルール
- UC-03: 現場作業を開始・終了する
- 境界コンテキスト「勤怠管理」ルール#2, #3, #4, #5
- 境界コンテキスト「現場管理」ルール#16, #17, #18

## 受入基準

### 現場開始
- [ ] 勤務中（出勤済み）の状態で [現場開始] を押すと、W-02（案件選択画面）に遷移する
- [ ] W-02に当日の担当案件がハイライト表示される
- [ ] 案件をタップすると詳細（住所、鍵情報、作業指示書）が表示される
- [ ] 住所をタップするとGoogleマップが起動する
- [ ] [作業開始] ボタンで現場開始が記録され、W-01に戻る
- [ ] ステータスが「勤務中」→「現場作業中（案件名）」に変わる
- [ ] 現場作業中は [現場開始] ボタンが非活性になる
- [ ] 現場作業中は [現場終了] ボタンが活性になる

### 案件選択画面（W-02）
- [ ] 当日の担当案件は上部にハイライト表示される
- [ ] 担当外の案件も選択可能（一覧表示）
- [ ] 各案件に物件名、住所、作業予定時間が表示される
- [ ] 案件詳細で鍵情報（受け取り方法＋備考）が確認できる
- [ ] FAX画像（作業指示書）が添付されている場合、拡大表示できる
- [ ] [← 戻る] でW-01に戻れる（選択せずに戻る）

### 現場終了
- [ ] 現場作業中の状態で [現場終了] を押すと、終了時刻が記録される
- [ ] ステータスが「現場作業中」→「勤務中」に戻る
- [ ] 作業時間（現場開始〜終了）が計算される
- [ ] 現場終了後、作業報告の促しが表示される
  - [報告する] → W-03（作業報告画面）に遷移
  - [後で] → 促しを閉じる。未提出バッジを表示

### 複数現場
- [ ] 1日に複数の現場を担当できる（現場A終了→現場B開始）
- [ ] 各現場の作業時間が個別に記録される

## 操作フロー

### 現場開始（正常系）

| # | ユーザー操作 | システム動作 |
|---|------------|------------|
| 1 | W-01で [現場開始] をタップ | — |
| 2 | — | W-02（案件選択画面）を表示 |
| 3 | — | 当日担当案件をハイライトで上部に表示 |
| 4 | 案件をタップ | — |
| 5 | — | 案件詳細を表示（住所、鍵情報、作業指示書） |
| 6 | [作業開始] をタップ | — |
| 7 | — | TimeStamp { type: 'SiteStart', time: now(), jobId } を保存 |
| 8 | — | DailyRecord.status を 'onSite' に更新 |
| 9 | — | W-01に戻り、ステータスを「現場作業中（案件名）」に更新 |

### 現場終了（正常系）

| # | ユーザー操作 | システム動作 |
|---|------------|------------|
| 1 | W-01で [現場終了] をタップ | — |
| 2 | — | TimeStamp { type: 'SiteEnd', time: now(), jobId } を保存 |
| 3 | — | 作業時間を計算（SiteStart〜SiteEnd） |
| 4 | — | DailyRecord.status を 'working' に更新 |
| 5 | — | 作業報告の促しを表示 |

## データモデル

SPEC-002で定義した `TimeStamp` / `DailyRecord` を使用。
現場作業では `jobId` フィールドを使って案件と紐付ける。

```javascript
// 案件選択画面で表示するデータ
JobListItem {
  jobId: string,
  propertyName: string,     // 物件名（例: "○○マンション 301号室"）
  address: string,          // 住所
  scheduledDate: string,    // 作業予定日
  assignedWorkers: string[],// 担当作業員
  isAssigned: boolean,      // このユーザーが担当か（ハイライト用）
  keyInfo: KeyInfo | null,  // 鍵情報
  instructionImage: string | null  // 作業指示書画像URL
}

// 鍵情報
KeyInfo {
  method: string,           // 受け取り方法（選択式）
  note: string              // 鍵備考（自由入力）
}
```

## UI仕様

### W-02: 案件選択画面レイアウト
- **ヘッダー**: [← 戻る] + 「案件を選択」タイトル
- **当日担当エリア**（ハイライト）: 担当案件カード一覧
- **全案件エリア**: その他の案件カード一覧
- **各カード表示内容**: 物件名、住所、作業予定時間
- **案件詳細（タップ時）**: 鍵情報、作業指示書画像、[作業開始] ボタン、住所（タップでGoogleマップ起動）
- **フッター**: [現場が一覧にない] ボタン → 仮登録（Phase 2 SPEC-007）
