## エンティティと値オブジェクトの分類

| 概念 | 種類 | 説明 |
|------|------|------|
| Account | エンティティ | ログイン情報。ユーザーを一意に識別。パスワードは管理者が一括管理 |
| Worker | エンティティ | 作業員の情報 |
| Property | エンティティ | 物件（建物・場所）の継続的な情報。住所、棟名、部屋番号、リピートフラグ |
| Job | エンティティ | 案件。物件に対する個別の清掃作業。作業予定日、担当者、鍵情報を持つ |
| DailyRecord | 集約ルート | 1日分の打刻記録のまとまり。管理者が確定した勤務時間を持つ |
| TimeStamp | エンティティ | 個々の打刻記録。jobId（旧siteId）で案件と紐づく |
| WorkReport | エンティティ | 作業報告。案件単位。評価ランク・評価備考を含む |
| PaySlip | エンティティ | 給与明細。管理者が登録したPDFファイル |
| Location | 値オブジェクト | GPS座標（lat, lng） |
| WorkStatus | 値オブジェクト | 勤務ステータス（未出勤/勤務中/現場作業中/退勤済み） |
| JobStatus | 値オブジェクト | 案件ステータス（未着手/作業中/完了/案件情報未確定） |
| CleaningArea | 値オブジェクト | 担当箇所（台所、浴室等） |
| TranslatedText | 値オブジェクト | 翻訳付きテキスト（original, originalLang, translated, translatedLang） |
| KeyInfo | 値オブジェクト | 鍵情報（keyPickupMethod + keyNote） |
| EvaluationRank | 値オブジェクト | 作業評価ランク |
| RepeatFlag | 値オブジェクト | リピートフラグ（boolean） |

## Site → Property + Job 分離の背景（2026-07-15 MTG）

旧モデルでは `Site`（現場）が1つのエンティティだったが、MTGにて以下が判明：

1. **同じ建物でも部屋番号が違えば別案件**
2. **リピート案件**がある（同じ物件に再度依頼が入る）
3. **物件データは蓄積**したいが、**FAX画像等は案件完了後に削除可能**

これにより、**継続的に保存する情報（Property）** と **作業日ごとに発生する情報（Job）** を分けて管理する設計に変更した。
