## 開発優先度

| 優先度 | コンテキスト | MVP | 理由 |
|:---:|------------|:---:|------|
| ★★★ | 勤怠管理（Attendance） | ✅ | プロジェクトの主目的 |
| ★★★ | 認証・アカウント（Auth） | ✅ | ロール分岐に必須 |
| ★★☆ | 現場管理（SiteManagement） | — | 物件/案件管理が勤怠と密接。打刻の次に重要 |
| ★★☆ | 多言語対応（i18n） | — | ユーザーの半数以上が外国人 |
| ★☆☆ | 人事・評価（HR） | — | 将来フェーズ（ただしランク付け機能は早期実装を検討） |
| ★☆☆ | 帳票・請求（Billing） | — | 将来フェーズ |

## コンテキスト間のデータフロー

| 送信元 | 送信先 | データ | タイミング |
|--------|--------|--------|-----------| 
| 認証 → 勤怠管理 | ログインユーザー情報 | ログイン時 |
| 認証 → 現場管理 | ログインユーザー情報 | ログイン時 |
| 勤怠管理 → 現場管理 | 打刻データ（現場開始/終了） | 打刻時 |
| 現場管理 → 勤怠管理 | 案件情報（物件名、住所、案件ID） | 現場選択時 |
| 勤怠管理 → 人事・評価 | 勤務実績データ、確定勤務時間 | 月次集計時 |
| 現場管理 → 人事・評価 | 作業実績、評価ランクデータ | 評価実施時 |
| 現場管理 → 帳票・請求 | 作業実績データ | 月次集計時 |
| 多言語対応 → 全コンテキスト | 翻訳テキスト | 画面表示時 |
| 人事・評価 → 作業員 | 給与明細PDF | 管理者登録時 |
