# 境界づけられたコンテキスト定義（Bounded Context Definition）

> 各コンテキストの責務・ビジネスルール・外部連携を定義する。
> 保留事項は `⏸️ 保留` と記載。解決後に更新する。
>
> **更新履歴**: 2026-07-15 MTG結果を反映。

---

# ★★★ 勤怠管理（Attendance）

| 項目 | 内容 |
|------|------|
| **目的** | 作業員の出勤・退勤・現場作業の時刻を正確に記録し、勤務実績を管理する |
| **主なアクター** | 作業員、管理者 |
| **開発優先度** | ★★★（MVP） |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| Worker（作業員） | エンティティ | 打刻を行う主体。名前、言語、連絡先、所属チームを持つ |
| TimeStamp（打刻） | エンティティ | 出勤/退勤/現場開始/現場終了 の記録 |
| WorkStatus（勤務ステータス） | 値オブジェクト | 未出勤 / 勤務中 / 現場作業中 / 退勤済み |
| DailyRecord（日次記録） | 集約ルート | ある作業員の1日分の打刻をまとめたもの。管理者確定の勤務時間を含む |
| Location（位置情報） | 値オブジェクト | 緯度・経度。打刻時に記録（任意） |
| ConfirmedHours（確定勤務時間） | 値オブジェクト | 管理者が確認・確定した実際の勤務時間 |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | 出勤していない状態で退勤はできない | ✅ 確定 |
| 2 | 現場開始は出勤後にのみ可能 | ✅ 確定 |
| 3 | 現場終了は現場開始後にのみ可能 | ✅ 確定 |
| 4 | 現場作業中に退勤しようとした場合、先に現場終了を促す | ✅ 確定 |
| 5 | 1日に複数の現場を担当できる（現場A終了→現場B開始） | ✅ 確定 |
| 6 | ⏸️ 休憩時間の扱い（打刻が必要か、勤務時間からの差し引き方法） | 保留 |
| 7 | ⏸️ 打刻修正の権限 → ただし管理者が勤務時間を確定できるため実質的に管理者が修正可能 | 保留（部分解決） |
| 8 | GPS位置情報の取得を拒否された場合、位置情報なしで打刻する | ✅ 確定 |
| 9 | 管理者もworkerロールと同様に自分の出退勤を打刻できる | ✅ 確定 |
| 10 | 共有端末でも自分のアカウントでログインすれば打刻できる | ✅ 確定 |
| 11 | 打刻は出勤/現場開始/現場終了/退勤の4ボタン方式。勤務プラン選択式は不採用 | ✅ 確定（2026-07-15 MTG） |
| 12 | 管理者はA-01ダッシュボードで各作業員の勤務時間を確定できる（給与計算に直結） | ✅ 確定（2026-07-15 MTG） |
| 13 | 残業時間は30分単位で管理する | ✅ 確定（2026-07-15 MTG） |
| 14 | 手当・交通費はアプリ上で管理しない | ✅ 確定（2026-07-15 MTG） |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 認証 → 勤怠管理 | 受信 | ログインユーザー情報（誰が打刻するか） |
| 現場管理 → 勤怠管理 | 受信 | 案件情報（現場開始時にどの案件かを選択） |
| 勤怠管理 → 人事・評価 | 送信 | 勤務実績データ（月次集計）、確定勤務時間 |
| 勤怠管理 → 帳票・請求 | 送信 | 勤務時間データ |

# ★★★ 認証・アカウント（Auth）

| 項目 | 内容 |
|------|------|
| **目的** | ユーザーの認証とロールに基づくアクセス制御 |
| **主なアクター** | 全ユーザー、管理者（アカウント管理） |
| **開発優先度** | ★★★（MVP） |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| Account（アカウント） | エンティティ | ログイン情報。userId, password, role, language |
| Role（ロール） | 値オブジェクト | worker / admin |
| DisplayLanguage（表示言語） | 値オブジェクト | ja / si / en |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | ログインは英数字パスワードで行う | ✅ 確定 |
| 2 | 1人のユーザーに1つのアカウントが紐づく（端末ではなく個人に紐づく） | ✅ 確定 |
| 3 | ロールは worker と admin の2種類 | ✅ 確定 |
| 4 | adminロールのユーザーはworkerの全機能も使用できる | ✅ 確定 |
| 5 | ログイン後、ロールに応じた画面を表示する（worker→打刻画面、admin→ダッシュボード） | ✅ 確定 |
| 6 | 表示言語はアカウントの language 設定に従う | ✅ 確定 |
| 7 | アカウントの作成・編集・削除は管理者のみが行える | ✅ 確定 |
| 8 | 共有端末でログアウト→別アカウントでログインが可能 | ✅ 確定 |
| 9 | パスワードの管理（変更・リセット）は管理者のみが行う。作業員にはパスワード変更機能を提供しない | ✅ 確定（2026-07-15 MTG） |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 認証 → 勤怠管理 | 送信 | ログインユーザー情報 |
| 認証 → 現場管理 | 送信 | ログインユーザー情報 |
| 認証 → 多言語対応 | 送信 | 表示言語設定 |

# ★★☆ 現場管理（SiteManagement）

| 項目 | 内容 |
|------|------|
| **目的** | 物件情報・案件情報・作業指示書・作業報告を一元管理する |
| **主なアクター** | 管理者（登録・割り振り）、作業員（報告・現場選択） |
| **開発優先度** | ★★☆ |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| Property（物件） | エンティティ | 建物・場所の継続的な情報。住所、棟名、部屋番号、リピートフラグ |
| Job（案件） | エンティティ | 物件に対する個別の清掃作業。作業予定日、担当者、鍵情報、作業指示書 |
| WorkReport（作業報告） | エンティティ | 案件完了後の報告。写真・備考・評価ランクを含む |
| CleaningArea（担当箇所） | 値オブジェクト | 台所、浴室、リビングなど。管理者が拡張可能 |
| TranslatedText（翻訳付きテキスト） | 値オブジェクト | 備考の原文＋翻訳 |
| KeyInfo（鍵情報） | 値オブジェクト | 鍵受け取り方法 ＋ 鍵備考 |
| InstructionImage（指示書画像） | 値オブジェクト | FAXの画像ファイル（JPG/PNG/PDF） |
| JobStatus（案件ステータス） | 値オブジェクト | 未着手/作業中/完了/案件情報未確定 |
| EvaluationRank（作業評価ランク） | 値オブジェクト | 管理者が物件単位で付与するランク |
| RepeatFlag（リピートフラグ） | 値オブジェクト | 管理者が付与する繰り返し物件の印 |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | 物件の登録・編集・削除は管理者のみが行える | ✅ 確定 |
| 2 | 1つの案件に複数の作業員を割り当てられる | ✅ 確定 |
| 3 | 作業報告には証拠写真を添付できる。写真ごとにコメントを付けられる | ✅ 確定（2026-07-15 MTG） |
| 4 | 作業報告は現場終了後に作成する | ✅ 確定 |
| 5 | 備考欄の自由入力は翻訳API（Google Cloud Translation）で自動翻訳される | ✅ 確定 |
| 6 | 翻訳時、原文（シンハラ語等）も保持する | ✅ 確定 |
| 7 | 作業指示書（FAX）は画像として案件に添付できる。JPG/PNG/PDF対応 | ✅ 確定（2026-07-15 MTG） |
| 8 | 作業指示書画像は案件完了後に一定期間で自動削除する。管理者が手動削除も可能 | ✅ 確定（2026-07-15 MTG） |
| 9 | 物件データ（住所等）は継続保存。案件データ（FAX画像等）は作業日ごとに発生 | ✅ 確定（2026-07-15 MTG） |
| 10 | 同じ建物でも部屋番号が異なれば別の物件（選択肢）として扱う | ✅ 確定（2026-07-15 MTG） |
| 11 | リピート物件には管理者がフラグを付けられ、一覧で絞り込みができる | ✅ 確定（2026-07-15 MTG） |
| 12 | 管理者向け物件一覧は全件表示ではなく、リピート物件＋最新50件程度をページネーションで表示 | ✅ 確定（2026-07-15 MTG） |
| 13 | 古い物件は削除せず非表示にし、検索した場合のみ表示する | ✅ 確定（2026-07-15 MTG） |
| 14 | 未登録現場の場合、作業員は「現場が一覧にない」を選択し仮登録できる。管理者が後から紐付け | ✅ 確定（2026-07-15 MTG） |
| 15 | 管理者はスマートフォン/管理画面から当日の案件を緊急追加できる（必須項目のみで登録） | ✅ 確定（2026-07-15 MTG） |
| 16 | 鍵情報は案件ごとに登録。受け取り方法（選択式）＋鍵備考（自由入力）のセット | ✅ 確定（2026-07-15 MTG） |
| 17 | 現場選択画面で住所をタップするとGoogleマップが起動する | ✅ 確定（2026-07-15 MTG） |
| 18 | 当日の担当案件は作業予定日を基準にハイライト/上部表示する | ✅ 確定（2026-07-15 MTG） |
| 19 | 担当箇所は管理者側が拡張できる | ✅ 確定（2026-07-15 MTG） |
| 20 | 完了報告用写真は不要だが、破損等の証拠写真は作業報告に添付する | ✅ 確定（2026-07-15 MTG） |
| 21 | 管理者が作業後の現場確認時に物件単位でランク付けし、ランク理由を備考に記録できる | ✅ 確定（2026-07-15 MTG） |
| 22 | 作業報告の差し戻し機能はアプリに実装しない | ✅ 確定（2026-07-15 MTG） |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 現場管理 → 勤怠管理 | 送信 | 案件情報（物件名、住所、案件ID） |
| 勤怠管理 → 現場管理 | 受信 | 打刻データ（現場開始/終了の時刻） |
| 現場管理 → 帳票・請求 | 送信 | 作業実績データ |
| 現場管理 → 人事・評価 | 送信 | 作業評価ランク、作業実績データ |

# ★★☆ 多言語対応（i18n）

| 項目 | 内容 |
|------|------|
| **目的** | UIを複数言語で表示し、自由入力テキストを翻訳する |
| **主なアクター** | システム（自動） |
| **開発優先度** | ★★☆ |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| TranslationFile（翻訳ファイル） | 設定 | 各言語のUIラベル翻訳（JSON） |
| TranslationAPI（翻訳API） | 外部サービス | Google Cloud Translation |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | UIの表示言語はアカウントの language 設定に従う | ✅ 確定 |
| 2 | 対応言語は日本語（ja）、シンハラ語（si）、英語（en）の3言語 | ✅ 確定 |
| 3 | 作業員（worker）のデフォルト言語はシンハラ語 | ✅ 確定 |
| 4 | 管理者（admin）のデフォルト言語は日本語 | ✅ 確定 |
| 5 | 自由入力テキストは送信時にAPIで翻訳し、原文と翻訳文の両方を保存する | ✅ 確定 |
| 6 | 翻訳APIが利用できない場合、原文のみで保存する（エラーにしない） | ✅ 確定 |
| 7 | 作業指示書（FAX）は日本語のままで問題ない（作業員は教育済み） | ✅ 確定（2026-07-15 MTG） |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 認証 → 多言語対応 | 受信 | ユーザーの表示言語設定 |
| 多言語対応 → 全コンテキスト | 送信 | 翻訳済みUIテキスト |
| 現場管理 → 多言語対応 | 送信 | 自由入力テキスト（翻訳リクエスト） |

# ★☆☆ 人事・評価（HR）

| 項目 | 内容 |
|------|------|
| **目的** | 勤務実績に基づく人事考課・給与計算を支援する |
| **主なアクター** | 管理者 |
| **開発優先度** | ★☆☆（将来フェーズ） |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| PerformanceReview（人事考課） | エンティティ | 作業員の評価記録 |
| EvaluationRank（作業評価ランク） | 値オブジェクト | 物件単位のランク付け |
| EvaluationNote（評価備考） | 値オブジェクト | ランクの理由を記録 |
| PaySlip（給与明細） | エンティティ | 管理者が登録するPDF形式の給与明細 |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | 人事考課は管理者のみが作成・編集できる | ✅ 確定 |
| 2 | 評価項目は安全・速さ・品質・対応可能な作業の幅 | ✅ 確定（2026-07-15 MTG） |
| 3 | 管理者が作業後の現場確認時に物件単位でランク付けを実施 | ✅ 確定（2026-07-15 MTG） |
| 4 | ランクの理由を備考欄に記録できる | ✅ 確定（2026-07-15 MTG） |
| 5 | 勤怠管理コンテキストから勤務実績・確定勤務時間データを受け取る | ✅ 確定 |
| 6 | 作業時間、差戻し回数、担当者、評価ランクを組み合わせて相対評価に活用 | ✅ 確定（2026-07-15 MTG） |
| 7 | 事前の難易度判断は困難（広さだけでなく汚れ具合にも左右されるため） | ✅ 確定（2026-07-15 MTG） |
| 8 | 給与計算は毎月10日締め。Excel運用を継続 | ✅ 確定（2026-07-15 MTG） |
| 9 | 給与明細はExcelで作成→PDF化。アプリでは管理者がPDFを登録し、作業員が自分の明細を閲覧可能 | ✅ 確定（2026-07-15 MTG） |
| 10 | 手当・交通費はアプリ上で管理しない | ✅ 確定（2026-07-15 MTG） |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 勤怠管理 → 人事・評価 | 受信 | 勤務実績データ（月次）、確定勤務時間 |
| 現場管理 → 人事・評価 | 受信 | 作業実績・評価ランクデータ |

# ★☆☆ 帳票・請求（Billing）

| 項目 | 内容 |
|------|------|
| **目的** | 作業実績に基づく請求書作成・収益性分析を支援する |
| **主なアクター** | 管理者 |
| **開発優先度** | ★☆☆（将来フェーズ） |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| Invoice（請求書） | エンティティ | 不動産会社向けの請求書 |
| ProfitAnalysis（収益性分析） | 値オブジェクト | 現場ごとの収益性 |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | 請求書の作成・発行は管理者のみが行える | ✅ 確定 |
| 2 | 作業実績から自動で請求額を計算する | ✅ 確定 |
| 3 | 現場ごとの収益性（売上 - 人件費）を算出できる | ✅ 確定 |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 勤怠管理 → 帳票・請求 | 受信 | 勤務時間データ、確定勤務時間 |
| 現場管理 → 帳票・請求 | 受信 | 作業実績データ |
