# 境界づけられたコンテキスト定義（Bounded Context Definition）

> 各コンテキストの責務・ビジネスルール・外部連携を定義する。
> 保留事項は `⏸️ 保留` と記載。解決後に更新する。

---

# ★★★ 勤怠管理（Attendance）

| 項目 | 内容 |
|------|------|
| **目的** | 作業員の出勤・退勤・現場作業の時刻を正確に記録し、勤務実績を管理する |
| **主なアクター** | 作業員、管理者 |
| **開発優先度** | ★★★（MVP） |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| Worker（作業員） | エンティティ | 打刻を行う主体。名前、言語、連絡先を持つ |
| TimeStamp（打刻） | エンティティ | 出勤/退勤/現場開始/現場終了 の記録 |
| WorkStatus（勤務ステータス） | 値オブジェクト | 未出勤 / 勤務中 / 現場作業中 / 退勤済み |
| DailyRecord（日次記録） | 集約ルート | ある作業員の1日分の打刻をまとめたもの |
| Location（位置情報） | 値オブジェクト | 緯度・経度。打刻時に記録（任意） |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | 出勤していない状態で退勤はできない | ✅ 確定 |
| 2 | 現場開始は出勤後にのみ可能 | ✅ 確定 |
| 3 | 現場終了は現場開始後にのみ可能 | ✅ 確定 |
| 4 | 現場作業中に退勤しようとした場合、先に現場終了を促す | ✅ 確定 |
| 5 | 1日に複数の現場を担当できる（現場A終了→現場B開始） | ✅ 確定 |
| 6 | ⏸️ 休憩時間の扱い（打刻が必要か、勤務時間からの差し引き方法） | 保留 |
| 7 | ⏸️ 打刻の修正権限（打ち忘れ時に誰が修正できるか） | 保留 |
| 8 | GPS位置情報の取得を拒否された場合、位置情報なしで打刻する | ✅ 確定 |
| 9 | 管理者もworkerロールと同様に自分の出退勤を打刻できる | ✅ 確定 |
| 10 | 共有端末でも自分のアカウントでログインすれば打刻できる | ✅ 確定 |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 認証 → 勤怠管理 | 受信 | ログインユーザー情報（誰が打刻するか） |
| 現場管理 → 勤怠管理 | 受信 | 現場情報（現場開始時にどの現場かを選択） |
| 勤怠管理 → 人事・評価 | 送信 | 勤務実績データ（月次集計） |
| 勤怠管理 → 帳票・請求 | 送信 | 勤務時間データ |

# ★★★ 認証・アカウント（Auth）

| 項目 | 内容 |
|------|------|
| **目的** | ユーザーの認証とロールに基づくアクセス制御 |
| **主なアクター** | 全ユーザー、管理者（アカウント管理） |
| **開発優先度** | ★★★（MVP） |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| Account（アカウント） | エンティティ | ログイン情報。userId, password, role, language |
| Role（ロール） | 値オブジェクト | worker / admin |
| DisplayLanguage（表示言語） | 値オブジェクト | ja / si / en |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | ログインは英数字パスワードで行う | ✅ 確定 |
| 2 | 1人のユーザーに1つのアカウントが紐づく（端末ではなく個人に紐づく） | ✅ 確定 |
| 3 | ロールは worker と admin の2種類 | ✅ 確定 |
| 4 | adminロールのユーザーはworkerの全機能も使用できる | ✅ 確定 |
| 5 | ログイン後、ロールに応じた画面を表示する（worker→打刻画面、admin→ダッシュボード） | ✅ 確定 |
| 6 | 表示言語はアカウントの language 設定に従う | ✅ 確定 |
| 7 | アカウントの作成・編集・削除は管理者のみが行える | ✅ 確定 |
| 8 | 共有端末でログアウト→別アカウントでログインが可能 | ✅ 確定 |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 認証 → 勤怠管理 | 送信 | ログインユーザー情報 |
| 認証 → 現場管理 | 送信 | ログインユーザー情報 |
| 認証 → 多言語対応 | 送信 | 表示言語設定 |

# ★★☆ 現場管理（SiteManagement）

| 項目 | 内容 |
|------|------|
| **目的** | 物件情報・作業依頼・作業報告を一元管理する |
| **主なアクター** | 管理者（登録・割り振り）、作業員（報告） |
| **開発優先度** | ★★☆ |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| Site（現場） | エンティティ | 清掃を行う物件。住所、依頼元、清掃難易度を持つ |
| WorkOrder（作業依頼） | エンティティ | 不動産会社からの依頼内容 |
| WorkReport（作業報告） | エンティティ | 作業完了後の報告。写真・備考を含む |
| CleaningArea（担当箇所） | 値オブジェクト | 台所、浴室、リビングなど |
| TranslatedText（翻訳付きテキスト） | 値オブジェクト | 備考の原文＋翻訳 |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | 現場の登録・編集・削除は管理者のみが行える | ✅ 確定 |
| 2 | 1つの現場に複数の作業員を割り当てられる | ✅ 確定 |
| 3 | 作業報告には写真を最大10枚添付できる | ✅ 確定 |
| 4 | 作業報告は現場終了後に作成する | ✅ 確定 |
| 5 | 備考欄の自由入力は翻訳API（Google Cloud Translation）で自動翻訳される | ✅ 確定 |
| 6 | 翻訳時、原文（シンハラ語等）も保持する | ✅ 確定 |
| 7 | 作業依頼はFAXで受領（将来的に電子化） | ✅ 確定 |
| 8 | ⏸️ 1日あたりの現場数の上限（設計に影響） | 保留 |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 現場管理 → 勤怠管理 | 送信 | 現場情報（名前、住所） |
| 勤怠管理 → 現場管理 | 受信 | 打刻データ（現場開始/終了の時刻） |
| 現場管理 → 帳票・請求 | 送信 | 作業実績データ |

# ★★☆ 多言語対応（i18n）

| 項目 | 内容 |
|------|------|
| **目的** | UIを複数言語で表示し、自由入力テキストを翻訳する |
| **主なアクター** | システム（自動） |
| **開発優先度** | ★★☆ |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| TranslationFile（翻訳ファイル） | 設定 | 各言語のUIラベル翻訳（JSON） |
| TranslationAPI（翻訳API） | 外部サービス | Google Cloud Translation |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | UIの表示言語はアカウントの language 設定に従う | ✅ 確定 |
| 2 | 対応言語は日本語（ja）、シンハラ語（si）、英語（en）の3言語 | ✅ 確定 |
| 3 | 作業員（worker）のデフォルト言語はシンハラ語 | ✅ 確定 |
| 4 | 管理者（admin）のデフォルト言語は日本語 | ✅ 確定 |
| 5 | 自由入力テキストは送信時にAPIで翻訳し、原文と翻訳文の両方を保存する | ✅ 確定 |
| 6 | 翻訳APIが利用できない場合、原文のみで保存する（エラーにしない） | ✅ 確定 |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 認証 → 多言語対応 | 受信 | ユーザーの表示言語設定 |
| 多言語対応 → 全コンテキスト | 送信 | 翻訳済みUIテキスト |
| 現場管理 → 多言語対応 | 送信 | 自由入力テキスト（翻訳リクエスト） |

# ★☆☆ 人事・評価（HR）

| 項目 | 内容 |
|------|------|
| **目的** | 勤務実績に基づく人事考課・給与計算を支援する |
| **主なアクター** | 管理者 |
| **開発優先度** | ★☆☆（将来フェーズ） |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| PerformanceReview（人事考課） | エンティティ | 作業員の評価記録 |
| CleaningQuality（清掃品質） | 値オブジェクト | 清掃の仕上がり評価 |
| CleaningSpeed（清掃スピード） | 値オブジェクト | 作業にかかった時間 |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | 人事考課は管理者のみが作成・編集できる | ✅ 確定 |
| 2 | 評価項目はスピード、品質、担当箇所の難易度 | ✅ 確定 |
| 3 | 清掃難易度が高い現場での実績は評価で考慮される | ✅ 確定 |
| 4 | 勤怠管理コンテキストから勤務実績データを受け取る | ✅ 確定 |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 勤怠管理 → 人事・評価 | 受信 | 勤務実績データ（月次） |
| 現場管理 → 人事・評価 | 受信 | 作業実績・品質データ |

# ★☆☆ 帳票・請求（Billing）

| 項目 | 内容 |
|------|------|
| **目的** | 作業実績に基づく請求書作成・収益性分析を支援する |
| **主なアクター** | 管理者 |
| **開発優先度** | ★☆☆（将来フェーズ） |

## このコンテキストに含まれるもの

| 概念 | 種類 | 説明 |
|------|------|------|
| Invoice（請求書） | エンティティ | 不動産会社向けの請求書 |
| ProfitAnalysis（収益性分析） | 値オブジェクト | 現場ごとの収益性 |

## ビジネスルール

| # | ルール | 状態 |
|---|--------|------|
| 1 | 請求書の作成・発行は管理者のみが行える | ✅ 確定 |
| 2 | 作業実績から自動で請求額を計算する | ✅ 確定 |
| 3 | 現場ごとの収益性（売上 - 人件費）を算出できる | ✅ 確定 |

## 外部との関係

| 連携先 | データの流れ | 内容 |
|--------|------------|------|
| 勤怠管理 → 帳票・請求 | 受信 | 勤務時間データ |
| 現場管理 → 帳票・請求 | 受信 | 作業実績データ |
