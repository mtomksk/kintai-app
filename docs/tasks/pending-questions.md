- [x] **現場までの移動方法** — 電車？車？直行直帰？事務所集合？（No.8）
  - → 9割車、1割電車。事務所集合後に現場へ移動。作業後は基本直帰（2026-07-15 MTG）
- [x] **鍵の受け取り方法** — 現場の鍵はどこで誰から受け取るか？（No.8）
  - → 9割：現場設置（ダイヤル式/暗証番号）、1割：管理者が顧客から事前受領し当日渡し。FAXの作業指示書に鍵情報が記載されている場合あり（2026-07-15 MTG）

### 打刻・勤怠

- [x] **打刻の方式** — リアルタイム打刻ではなく、シフト選択式（フル出勤/午前/午後）か？（No.6）
  - → 勤務プラン（選択式）は不採用。出勤/現場開始/現場終了/退勤の4ボタンを維持。残業もあるため柔軟性が必要。残業は30分単位（2026-07-15 MTG）
- [x] **シフト選択式の場合の給与計算** — 各シフトに固定時間を割り当てるか？（No.6）
  - → 選択式は不採用。代わりにA-01管理ダッシュボードで管理者が実際の勤務時間を選択/入力して確定できるようにする（打刻間違い・嘘のカバー＋給与計算直結）（2026-07-15 MTG）
- [x] **手当・交通費の入力** — アプリで入力する必要があるか？（No.21）
  - → アプリ上での交通費管理は不要。手当は変動するため自動反映は困難。車の燃料代程度なので不要（2026-07-15 MTG）
- [x] **給与計算の出力先** — Excelのみで計算するか、管理画面にも表示するか？（No.22）
  - → 給与計算は毎月10日締めでExcel運用を継続。完成した給与明細PDF（現在LINEで送付）をアプリに登録し、従業員が自分の明細を閲覧できる機能を実装。初期段階ではPDFの登録・閲覧機能のみ（2026-07-15 MTG）

### 現場管理

- [x] **現場が未登録の場合** — スキップして打刻できるようにするか？（No.9）
  - → 単純スキップではなく「未登録現場として仮登録」方式。作業員が物件名/住所を簡易入力→管理者画面に「案件情報未確定」として表示→管理者が後から紐付け/新規登録（2026-07-15 MTG）
- [x] **現場と日付の紐づけ** — 日付ごとに現場一覧を出す方式にするか？（No.11）
  - → 作業予定日で当日の現場を上部表示/ハイライト。ただし案件IDで管理し、日付だけでは一意にしない。リピート物件にはフラグを付け、管理者画面で絞り込み可能（2026-07-15 MTG）

### 作業報告

- [x] **差し戻し機能** — 管理者が作業報告を差し戻す運用はあるか？どのような場合に差し戻すか？（No.2）
  - → 顧客→管理者の差し戻しはアプリ実装しない（現状運用を維持）。打刻誤りはA-01で管理者が勤務時間を確定することでカバー。報告の質に関する差し戻しは、意識付けの効果はあるが現実的に写真不足等は対応困難なため、まずは管理者が勤務時間を確定する機能を優先（2026-07-15 MTG）
- [x] **担当箇所の管理** — 現場ごとの担当箇所は添付ファイル（指示書等）で管理しているか？（No.13）
  - → 作業指示書（FAX）を画像として添付する機能を実装。JPG/PNG/PDF対応。スマホで拡大表示可能。案件完了後に一定期間で自動削除（2026-07-15 MTG）
- [x] **パスワードを忘れた場合** — 現在どのように対応しているか？（管理者がリセット？）（No.4）
  - → パスワード管理は管理者が一括管理。作業者にはパスワード変更機能を提供しない。セキュリティより工程削減を重視（2026-07-15 MTG）

### 人事評価

- [x] **新しい評価指標の追加** — 「報告の質」「難易度×成果の質×時間」といった指標を導入したいか？（No.24）
  - → 管理者が作業後の現場確認時に物件単位でランク付けを実施。ランク理由を記録する備考欄を設ける。重視項目は安全/速さ/品質/対応可能な作業の幅。完了報告用写真は不要だが証拠写真（破損等）は添付。まずは簡単なランク付けと記録機能から実装（2026-07-15 MTG）
