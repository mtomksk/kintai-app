---

## 1. プロジェクト概要

### 目的
清掃作業員の勤怠管理をデジタル化し、管理業務の効率化を図る。

### 背景
- 現在、LINE・口頭・FAX・カレンダーアプリ・Excelなど複数手段で勤怠や作業実績を管理
- 情報が分散しており、管理者の業務負担が大きい
- 清掃業は利益率が低く、業務効率化が重要

### ユーザー
| ロール | 人数 | 説明 |
|--------|------|------|
| 作業員（Worker） | 8名 | 清掃作業を行う。日本人2名、スリランカ人6名 |
| 管理者（Manager） | 2名 | 勤怠管理・現場割り振り・評価を行う |

### 特記事項
- 日本語を読めない作業員が多い（多言語対応が必須）
- スマートフォン未所持の作業員がいる（共有端末＋アカウント制）
- 日本語を話せるスリランカ人リーダー（マリク）が在籍

## 2. アーキテクチャ方針

### アプリ構成
- 1つのWebアプリケーション
- ログイン後、ロール（worker / admin）で画面を切替
- モバイルファースト（作業員はスマホで使用）

### 技術スタック
- フロントエンド: HTML + CSS + JavaScript（Vanilla）
- データ保存: （検討中 — localStorage → 将来的にバックエンド）
- 対応言語: 日本語 / シンハラ語 / 英語
- 翻訳API: Google Cloud Translation（自由入力テキストの翻訳用）

### 多言語対応の方針

| 対象 | 方法 | 説明 |
|------|------|------|
| UI表示（ボタン名、ラベル等） | i18n 翻訳ファイル（JSON） | 事前に用意した翻訳を表示。APIは不要 |
| 自由入力テキスト（備考欄等） | 翻訳API（Google Cloud Translation） | 送信時にAPIで自動翻訳。原文も保持する |

- UIの表示言語はアカウントの `language` 設定に従う
- 作業員（worker）: シンハラ語または英語で表示
- 管理者（admin）: 日本語で表示

### 制約
- フレームワークは使用しない（Vanilla JS）
- オフラインでも基本機能（打刻）が使えること
- レスポンシブデザイン（スマホ・タブレット・PC対応）

## 3. 命名規則

| 対象 | 規則 | 例 |
|------|------|---|
| CSS クラス | BEM記法 | `.clock-button__icon--active` |
| JavaScript 変数・関数 | camelCase | `calculateTotalHours` |
| JavaScript クラス | PascalCase | `TimeStampService` |
| ファイル名 | kebab-case | `work-report.js` |
| ドメイン用語 | ユビキタス言語集に従う | `docs/domain/ubiquitous-language.md` 参照 |

## 4. 開発フロー

1. **DDD** でドメインモデルを定義（`docs/domain/`）
2. **SDD** で機能仕様を作成（`docs/specs/`）
3. **AIエージェント**（Antigravity / Claude Code）が仕様に基づき実装
4. **レビュー** — 仕様と照合して検証
5. **Git** — ブランチ運用でチーム開発
